### 특수 기능 - `@Component, @Service`와 달리 `@Controller, @Repository`어노테이션은 지원하는 특수한 기능들이 더 존재한다. ### @Controller - **역할** : 웹 요청을 처리하는 Spring MVC의 컨트롤러로 동작하게 함 - 지원 기능 - `@RequestMapping`, `@GetMapping`, `@PostMapping` 같은 애노테이션으로 HTTP 요청을 매핑할 수 있게 됨. - 반환값이 `String`이면 뷰 이름으로 인식되어 템플릿 렌더링(`thymeleaf`, `jsp` 등)에 사용됨. - **`@ResponseBody` 없이 JSON 응답을 직접 반환하려면** `@RestController` 사용 (`@Controller + @ResponseBody` 조합). - 예외 처리: `@ExceptionHandler`, `@ControllerAdvice`와 결합해 요청 수준에서 예외 처리 가능. - 요약: HTTP 요청 처리, 뷰 렌더링, 요청 매핑, 예외 처리 기능 지원.`` ### @Repository - **역할**: 데이터 액세스 계층을 나타내며, 데이터베이스 작업을 담당. - **지원 기능**: - 예외 변환(AOP): **스프링이 JDBC 등의 기술에서 발생하는 예외를 자동으로 스프링의 `DataAccessException`으로 변환**해줌. - 예: `SQLException → DataAccessException` - 트랜잭션에서 예외가 발생해도 적절히 감지되고 롤백 처리 가능. - 스프링 데이터 JPA에서는 `JpaRepository`를 상속한 인터페이스에 대해, 스프링이 **구현체를 자동 생성**하고, `@Repository` 역할을 수행하는 **프록시 빈으로 등록**해줌 - 요약: 예외 변환(AOP), JPA 연동 최적화, 트랜잭션 에러 대응 등 **데이터 관련 기능** 강화.
