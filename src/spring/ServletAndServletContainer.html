## 서블릿(Servlet) - **정의**: 자바 기반 웹 애플리케이션에서 **클라이언트 요청(주로 HTTP 요청)**을 받아서 처리하고, **응답(HTTP 응답)**을 만들어 돌려주는 객체. - **역할**: 웹 브라우저가 보낸 요청을 받아 로직을 수행한 뒤, 결과를 HTML, JSON 등으로 응답한다. - **예시**: - `/test`라는 주소로 요청이 들어오면 `TestServlet`이 실행되어 `"test"`라는 응답을 내려줌. - **비유**: 식당에서 **주문을 받아서 요리를 만들어 손님에게 전달하는 서버 직원**. --- ## 서블릿 컨테이너(Servlet Container) - **정의**: 서블릿이 제대로 동작할 수 있도록 관리하고 실행 환경을 제공하는 프로그램. (대표적으로 **Tomcat, Jetty, Undertow** 등이 있음) - **역할**: 1. 서블릿 객체 생성 및 생명주기 관리 (init → service → destroy) 2. 요청과 응답 객체(HttpServletRequest, HttpServletResponse) 생성 후 서블릿에게 전달 3. 멀티스레드 기반으로 여러 요청을 동시에 처리 - **비유**: 식당의 **주방 시스템**. 직원(서블릿)이 요리를 할 수 있도록 주방, 재료, 주문 전달 시스템을 제공. → 직원(서블릿) 혼자서는 손님 응대를 할 수 없고, 반드시 주방 시스템(서블릿 컨테이너)이 필요함. --- ## 정리 - **서블릿**: 요청과 응답을 처리하는 자바 객체 (= 일하는 직원). - **서블릿 컨테이너**: 서블릿이 일할 수 있는 환경을 제공하고 관리하는 실행 도구 (= 주방 시스템). - 서블릿(Servlet) - 쉽게 말하면 **웹 요청을 처리하는 단위 객체**. - 스프링 부트로 치면 **Controller 하나하나**에 해당. - HTTP 요청을 받아서 처리하고, HTTP 응답을 돌려줌. - 자바 클래스 형태로 구현되며, `HttpServlet`을 상속해서 작성. - 서블릿 컨테이너(Servlet Container) - **서블릿이 요청을 처리할 수 있도록 중간에서 관리해주는 프로그램**. - 클라이언트의 HTTP 요청을 받고, 해당 요청을 알맞은 서블릿으로 전달. - 서블릿의 **생명주기 관리, 멀티스레드 요청 처리, 요청·응답 객체 생성** 등을 담당. - 대표적으로 **Tomcat, Jetty, Undertow** 등이 있음. - 스프링 부트에서는 내장 톰캣이 서블릿 컨테이너 역할을 함.
