### 쿠키 속성 - HttpOnly - Secure - SameSite ### HttpOnly - `HttpOnly`속성이 설정 된 쿠키는 JavaScript(`document.cookie`)를 통해 접근 불가 - 이를 통해 XSS(크로스 사이트 스크립팅) 공격으로부터 보호 - 하지만, 이 설정만으로는 CSRF(크로스 사이트 요청 위조)공격을 방어하지 못함 ### SameSite - 쿠키 방식의 취약점인 CSRF 공격 방어를 위해 주로 사용 - 적용되는 부분은 브라우저가 다른 사이트에서 쿠키 요청을 할 때 포함 여부 결정 - 지원되는 속성들 - `Strict` : 가장 강력한 보안, 오로지 같은 사이트에서만 쿠키를 전송 - 예로들어 A -> A 가능 / A -> B 불가능 / 다른 사이트에서 POST 요청 불가능 - 보안이 강력하기에 사용자의 경험 부분에서 불편 - `Lax` : 쿠키의 기본 값, A -> A 가능 / A -> B 가능, 다만 GET / HEAD 요청만 가능 - 안전한 요청(GET,HEAD) 방식에만 쿠키를 전송, POST, PUT, DELETE는 불가 - `none` : 모든 크로스 사이트 요청에서 쿠키 허용, 단 Secure 속성 설정 필수 ### Secure - `secure`가 설정 된 쿠키는 `HTTPS` 요청에서만 브라우저 전송 가능 - **HTTP 요청에서는 쿠키가 전송되지 않음** → 네트워크 스니핑 공격 방어 가능
