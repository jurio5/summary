```java static List direct(List numbers) { List result = new ArrayList<>(); for (Integer number : numbers) { if (number % 2 == 0) { result.add(number * 2); } } return result; } static List lambda(List numbers) { List filteredList = GenericFilter.filter(numbers, n -> n % 2 == 0); List result = GenericMapper.map(filteredList, x -> x * 2); return result; } ``` - `direct()` 와 `lambda()` 메서드는 서로 다른 프로그래밍 스타일을 보여줌 #### direct 메서드 - 프로그램을 어떻게 수행해야 하는지 수행 절차를 명시함 - 쉽게 이야기해서 개발자가 로직 하나하나를 어떻게 실행해야 하는지 명시 - 이런 프로그래밍 방식을 `명령형 프로그래밍`이라 함 - 명령형 스타일은 익숙하고 직관적이나, 로직이 복잡해질수록 반복 코드가 많아질 수 있음 #### lambda 메서드 - 무엇을 수행해야 하는지 원하는 결과에 초점을 맞춤 - 쉽게 이야기해서 특정 조건으로 필터하고, 변호나하라고 선언하면, 구체적인 부분은 내부에서 수행 - 개발자는 필터하고 변환하는 것 즉, 무엇을 해야 하는가에 초점을 맞춤 - 예를 들어 실제 어떻게 for문과 if문 등을 사용해서 필터하고 변환할지를 개발자가 크게 신경쓰지 않아도 된다. - 이런 프로그래밍 방식을 `선언형 프로그래밍`이라 함 - 선언형 스타일은 무엇을 하고자 하는지가 명확히 드러난다. 따라서 코드 가독성과 유지보수가 쉬워진다. - 여기서는 필터하고 변환하는 것에만 초점을 맞춘다. 실제 어떻게 필터하고 변환할지는 해당 기능을 사용하는 입장에서는 신경쓰지 않는다. # 명령형 vs 선언적 프로그래밍 ## 명령형 프로그래밍 (Imperative Programming) - **정의** 프로그램이 **어떻게(How)** 수행되어야 하는지를 명시하는 방식. 수행 절차와 단계들을 하나하나 기술한다. - **특징** - **단계별 실행**: 프로그램의 각 단계를 명확히 지정하고 순서대로 실행. - **상태 변화**: 변수 값 등 프로그램 상태의 변화를 명시. - **낮은 추상화**: 내부 구현을 직접 제어해야 하므로 추상화 수준이 낮음. - **예시** 전통적인 `for` 루프, `while` 루프 등을 명시적으로 작성하는 코드. - **장점** 시스템의 상태와 실행 흐름을 세밀하게 제어할 수 있음. --- ## 선언적 프로그래밍 (Declarative Programming) - **정의** 프로그램이 **무엇(What)** 을 수행해야 하는지를 명시하는 방식. 원하는 **결과**에 집중하고, 수행 방식은 추상화됨. - **특징** - **문제 해결에 집중**: 어떻게(how)가 아니라 무엇(what)을 원하는지에 초점. - **코드 간결성**: 더 간결하고 읽기 쉬운 코드 작성 가능. - **높은 추상화**: 내부 구현을 숨기고 원하는 결과에 집중. - **예시** `filter`, `map` 같은 고차 함수, HTML, SQL 등. - **장점** 코드가 간결하고 의도가 명확하며 유지보수가 쉬움. --- ## 정리 - **명령형 프로그래밍**: → 프로그램이 수행할 각 단계와 처리 과정을 상세히 기술, **어떻게** 결과에 도달할지를 명시. - **선언적 프로그래밍**: → 원하는 결과나 상태를 기술, **무엇**을 원하는지에 집중하며 내부 처리 방식은 추상화. - 특히 **람다**나 고차 함수를 사용하면 선언적 스타일로 코드를 더 간결하게 작성할 수 있다.
