- 기본적으로 코드 조각을 자바 문법으로 처리를 하려면 많이 힘이 듦 - 이유는 자바는 함수만 따로 사용할 수 있는 방법이 없음, 항상 객체 안에 메서드를 선언해서 객체와 메서드가 강하게 결합되어 있는 상태임 - 함수는 메서드의 넓은 단위임, 자바에서 객체 안에 선언하는 함수를 메서드라고 함 - C언어나 파이썬은 함수를 만들면, 바로 함수를 사용할 수 있지만 자바의 경우는 항상 사용하기 전 객체를 불러온 뒤 (객체).(메서드) 방식으로 사용해야 함 - 이러한 문제점을 해결하기 위해 자바8 버전 이후부터 패러다임이 바뀜, 람다가 출시된 것 - 아래는 간단한 람다의 사용 방법 (익명 클래스를 람다화) - MyFunction 는 apply 라는 메서드를 지원하는 간단한 인터페이스를 이용하여 람다로 구현 ```java public static void main(String[] args) { MyFunction myFunction = (int a, int b) -> { return a + b; }; int result = myFunction.apply(1, 2); System.out.println("result = " + result); } ``` ### 람다 정의 - 람다는 익명 함수이다. - 람다 역시 인스턴스(객체)이다. - 람다와 익명 클래스의 차이가 있지만, 익명 클래스를 편하게 바꿀 수 있다. - 람다는 코드를 간결하게 만드는 문법적 편의를 지칭하는 문법적 설탕(Syntactic sugar)이다. - 메서드나 함수는 다음과 같이 표현 ```java 반환타입 메서드명(매개변수) { 본문 } ``` - 람다는 다음과 같이 간결하게 표현 ```java (매개변수) -> {본문} ``` - 람다 vs 람다식 (Lambda Expression) - 람다 : 익명 함수를 지칭하는 일반적인 용어, 쉽게 이야기해서 개념을 뜻함 - 람다식 : (매개변수) -> {본문} 형태로 람다를 구현하는 구체적인 문법 표현을 지칭 - 쉽게 이야기해서 람다는 개념을, 람다식은 자바에서 그 개념을 구현하는 구체적인 문법을 의미, 람다가 넓은 의미
