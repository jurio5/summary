### JPQL - 가장 단순한 조회 방법 - `entityManager.find()` - 객체 그래프 탐색(`a.getB().getC()) - 나이가 18살 이상인 회원을 모두 검색하고 싶다면? - JPA를 사용하면 엔티티 객체를 중심으로 개발 - 문제는 검색 쿼리인데, 검색을 할때도 테이블이 아닌 엔티티 객체를 대상으로 검색을 해야 함 - 모든 DB 데이터를 객체로 변환해서 검색하는 것은 불가능 - 애플리케이션이 필요한 데이터만 DB에서 불러오려면 결국 검색 조건이 포함된 SQL이 필요 - JPA는 SQL을 추상화한 JPQL 이라는 객체 지향 쿼리 언어를 제공함 - SQL과 문법 유사, `SELECT, FROM, WHERE, GROUP BY, HAVING, JOIN` 지원 - JPQL 은 엔티티 객체를 대상으로 쿼리 - SQL 은 데이터베이스 테이블을 대상으로 쿼리 - 테이블이 아닌 객체를 대상으로 검색하는 객체 지향 쿼리 - SQL을 추상화해서 특정 데이터베이스 SQL에 의존하지 앟음 - JPQL을 한 마디로 정의하면 객체 지향 SQL 이라고 함 ### QueryDSL - 문자가 아닌 자바코드로 JPQL을 작성할 수 있음 - JPQL 빌더 역할 - 컴파일 시점에 문법 오류를 찾을 수 있음 (`type-safe`) - 동적 쿼리 작성이 편리함 - 단순하고 쉬움 - 실무 사용할 때 권장 ### 네이티브 SQL - 데이터베이스 고유의 SQL 문법을 그대로 사용하는 순수 쿼리 - JPA가 제공하는 SQL을 직접 사용하는 기능 - JPQL로 해결할 수 없는 특정 DB에 의존적인 기능 - 예로들면 오라클 `CONNECT BY`, 특정 DB만 사용하는 SQL 힌트 ### JDBC 직접 사용, SpringJdbcTemplate - JPA를 사용하면서 JDBC 커넥션을 직접 사용하거나, 스프링 JdbcTemplate, 마이바티스 등을 함께 사용 가능 - 단 영속성 컨텍스트를 적절한 시점에 강제로 플러시 필요 - 예로 들어 JPA를 우회해서 SQL을 실행하기 직전에 영속성 컨텍스트 수동 플러시 ```java Member member = new Member; member.setName("member1"); em.persist(member); // DB 정보가 없을 때 강제로 플러시가 필요하다는 부분 em.flush(); // 직업 flush 를 강제호출 해줘야 SQL이 DB에 날라감 // DB Connection 을 가져온 뒤 dbconn.executeQuery("select * from member); // 이 땐 영속성 컨텍스트 내부에만 있을 뿐 실제 DB에는 정보가 없음 // 위에서 flush 를 강제 호출해야 데이터를 조회해서 사용할 수 있음 ```
