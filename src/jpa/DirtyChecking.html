- JPA는 **트랜잭션이 끝날 때 변경된 엔티티를 자동으로 감지**하여 업데이트 ```java Member member = em.find(Member.class, 1L); member.setName("변경된 이름"); // 변경 감지 // 트랜잭션이 커밋될 때 자동으로 UPDATE 실행됨 tx.commit(); ``` - `setName()`을 호출해도 `em.persist()`를 다시 호출할 필요 없음 - 영속성 컨텍스트에 의해 트랜잭션 커밋 시점에 자동으로 `UPDATE` 쿼리 실행됨 - 변경 감지 (`Dirty Checking`) - 더티 체킹 대상이 된 엔티티 데이터가 있다면, 영속화를 진행하지 않아도 자동으로 JPA가 `update`를 해준다. - 더 자세하게 설명을 해보면, 영속성 컨텍스트 내부에서 데이터 변경이 일어날 때 그 엔티티는 더티 체킹의 대상이 되면서 `flush`될 때 기존의 영속성 컨텍스트를 찍어둔 스냅샷이 있다면 변경 된 데이터를 비교하여 자동으로 `update SQL`을 생성하고 DB로 `flush`를 한 뒤 `commit`을 하여 영구적으로 DB에 저장된다.
