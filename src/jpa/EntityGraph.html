```java class MemberRepository @Override @EntityGraph(attributePaths = {"team"}) List findAll(); @Query("select m from Member m") @EntityGraph(attributePaths = {"team"}) List findMemberEntityGraph(); // NamedEntityGraph 어노테이션을 통해 사용할 수 있지만 잘 사용안 함 // @EntityGraph("Member.all") @EntityGraph(attributePaths = {"team"}) List findEntityGraphByUsername(@Param("username") String username); ``` - `EntityGraph`는 정말 간단한 패치 조인을 할 때 사용하는 용도이고, 기본적으로 쿼리 메서드 방식을 사용하다가 조금 복잡해지면 패치 조인 방식을 채택하게 되는데 거기까지 간다면, 패치 조인 그 이상으로 해결을 하기에, `EntityGraph`를 많이 사용하진 않는다. 고로, 간단한 패치 조인을 사용할 때 사용된다.
