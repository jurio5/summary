- 특정 엔티티를 영속 상태로 만들 대 연관 된 엔티티도 함께 영속 상태로 만들고 싶을 때 사용 - 예로들면 부모 엔티티를 저장할 때 자식 엔티티도 함께 저장 ### 영속성 전이의 주의사항 - 영속성 전이는 연관관계를 매핑하는 것과 아무 관련이 없음 - 엔티티를 영속화 할 때 연관 된 엔티티도 함께 영속화하는 편리함을 제공할 뿐 - 영속성 전이를 아무 곳이나 막 걸어주는게 아니라 설계 판단을 한 뒤 걸어줘야 함 - 예로들면 `parent`와 `child`가 있으면, `parent`에서만 `child`를 사용하면 걸어도 괜찮은데 `parent`와 관련이 없는 다른 객체에서 `child`와 관계가 있다면 사용 X - 즉, 소유자가 `1`개 일 때 사용, `parent`와 `child`의 라이프 사이클이 동일할 때 사용 ### 영속성 전이 종류 - `ALL` : 모두 적용 - `PERSIST` : 영속 - `REMOVE` : 삭제 - `MERGE` : 병합 - `REFRESH` - `DETACH` - 보통은 `ALL`,`PERSIST`만 사용
