### 1. 관계의 핵심 - **FK를 이용해 테이블 간 연결** - DB의 관계에는 **방향이 없다** → `JOIN`으로 어느 쪽에서도 접근 가능 (양방향) - 단, 자바에선 서로 양방향 관계를 맺어야 서로의 도메인을 확인할 수 있음 → 객체 그래프 탐색 개념 때문에 ### 2. 관계의 핵심 속성 - **카디널리티(Cardinality)** : 1:1 / 1:N / N:M - **참여도(Optionality)** : 필수(Mandatory) / 선택(Optional) ### 3. 참여도 구현 | 비즈니스 규칙 | FK 제약 조건 | 의미 | | ---------------- | ---------- | ------ | | 회원은 팀에 소속될 수도 있다 | `NULL` 허용 | 선택적 관계 | | 회원은 반드시 팀에 소속 | `NOT NULL` | 필수적 관계 | ```sql CREATE TABLE member ( member_id BIGINT AUTO_INCREMENT PRIMARY KEY, team_id BIGINT NULL, name VARCHAR(50) NOT NULL, CONSTRAINT fk_member_team FOREIGN KEY (team_id) REFERENCES team (team_id) ); ``` - `NULL`을 허용함으로써 선택적 관계를 뜻함 ### 4. 까마귀발 표기법 (Crow’s Foot) - **선의 끝 기호는 반대편 엔티티의 규칙을 의미** - `O—|` : 선택적 참여 - `|—|` : 필수적 참여 - `|<` : 여러 개 관계 (다) ### 5. 일대다(1:N) 관계 설계 - 외래 키는 **항상 “다(N)” 쪽**에 둔다. (`연관 관계 주인`) → 예: `member.team_id → team.team_id` - 관계형 DB에서는 조인으로 양방향 탐색 가능
