### 조건 - **기본 키 제약 조건** - `null` 값을 허용하지 않아야 한다 (`NOT NULL`) - 중복되면 안 된다 (`UNIQUE`) - 값이 변경되면 안 된다 (불변성 유지) - 현실에서는 이러한 조건을 **영구적으로 만족하는 자연 키**를 찾기 어렵다. 예: 주민등록번호는 자연 키처럼 보이지만, 개인정보 보호 등의 이유로 변경되거나 사용이 제한될 수 있다. - 따라서 대부분의 경우 **대체 키(Surrogate Key)**를 사용하는 것이 좋다. - **자연 키**: 의미 있는 실제 데이터에서 유래한 키 (예: 주민등록번호, 이메일 등) - **대체 키**: DB 시스템이 자동으로 생성하고 관리하는 의미 없는 키 (예: 자동 증가 숫자, UUID 등) - 권장 : `Long`형 + 대체키 + 키 생성 전략 사용 (`GeneratedValue`, `UUID` 등)
